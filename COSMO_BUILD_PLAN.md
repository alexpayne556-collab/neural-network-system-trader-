# Cosmo build plan

## Phase 0 — truth substrate
- Build a persistent ledger for theses, evidence, kill conditions, and raw events.
- Make every claim traceable back to an evidence source.
- Treat the ledger as the system's memory and immune system.

## Phase 1 — ingestion and triage
- Add foragers that pull from SEC filings, news, and market data.
- Normalize every incoming event into the same schema.
- Run the event through the triage orchestrator before waking the swarm.

## Phase 2 — specialized swarms
- Bottleneck swarm for structural thesis plays.
- Biotech binary swarm for calendar and catalyst-driven plays.
- Cascade swarm for geopolitical and event-propagation logic.
- Microcap shock swarm for squeeze-style opportunities.

## Phase 3 — human override and learning loop
- Let the human edit, veto, or extend the thesis.
- Capture every override in the ledger.
- Grade outcomes after the fact and revise the thresholds.

## Phase 4 — orchestration and runtime
- Add an event bus and scheduler.
- Run the foragers continuously.
- Wake specialized swarms only when a signal crosses the threshold.

## Provider strategy
- Claude and Gemini should be used as reasoning providers, not as the whole system.
- Ollama or local models can be used for cheaper, repeatable tasks.
- API keys should be treated as expensive, high-value reasoning capacity rather than unlimited chat.
